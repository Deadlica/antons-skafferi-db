HTTP Request

GET

api/employee

HTTP Request: GET

Returns: all employees

api/employee?id=

HTTP Request: GET

Parameters:

@id: 12 digit social security number (ssn) of a registered employee.

Returns: the employee matching the ssn.

api/employee/available

HTTP Request: GET

Returns: all employees who are not working today.

api/employee/available?date=

HTTP Request: GET

Parameters:

@date: a given date following the format YYYY-MM-DD.

Returns: all employees who are not working the specified date.

api/employee

HTTP Request: POST

Body:

@Employee: {"email":"example@mail.com","firstName":"exampleName","lastName":"exampleLastName","phoneNumber":"0701234567","ssn":"199506239912"}.

Action: inserts a new employee.

api/employee

HTTP Request: PUT

Body:

@Employee: {"email":"example@mail.com","firstName":"exampleName","lastName":"exampleLastName","phoneNumber":"0701234567","ssn":"199506239912"}.

Action: deletes an employee.

api/shift

HTTP Request: GET

Param:

@employeeId(optional): 12 digit social security number (ssn) of a registered employee.

@shiftId(optional): id of a shift.

@date(optional): a date.

Returns: the parameters can be combined freely, will return shifts based on the combined parameters. For example if @employeeId and @date is set it will return all shifts for an employee on the given date.
